# Last 20-minute checklist

Run through this right before an LLD round.

## Patterns (60 seconds each — name + one sentence)

- [ ] Singleton — one instance
- [ ] Factory — which type to create
- [ ] Builder — step-by-step construction
- [ ] Strategy — swap algorithm
- [ ] Observer — notify on change
- [ ] State — behavior follows mode
- [ ] Decorator — wrap to add behavior
- [ ] Adapter — incompatible interface
- [ ] Facade — one door into subsystem
- [ ] Command — request as object

## Confusion pairs (30 seconds each)

- [ ] Strategy vs State
- [ ] Factory vs Abstract Factory
- [ ] Decorator vs Proxy
- [ ] Adapter vs Facade
- [ ] Template Method vs Strategy

## SOLID (one line each)

- [ ] S — one reason to change
- [ ] O — extend without modifying
- [ ] L — subtype substitutable
- [ ] I — small interfaces
- [ ] D — depend on abstractions

## Machine coding skeleton

Can you whiteboard in 2 minutes?

- [ ] Entities + enums
- [ ] Core service class
- [ ] One pattern named explicitly
- [ ] One extension point ("if pricing changes…")

## Mindset

- Clarify requirements first (5 min).
- Name patterns as you use them — interviewers want to hear *why*.
- Leave time for follow-ups (testing, concurrency, scale).
